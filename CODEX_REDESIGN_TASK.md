# Задание для Codex: редизайн сайта avtovyshki36.ru

**Цель:** обновить внешний вид и UX без изменения смысла контента и без поломки существующей логики (модалка, калькулятор, слайдер, cookie-consent, метрика).

Проект сейчас состоит из:
- `index.php` (главная)
- `header.php` (шапка)
- `footer.php` (подвал)
- `css/styles.css` (основные стили)
- (подключаются, но не в этом задании) `js/script.js`, `js/calc.js`, `css/form.css`, `js/sweetalert.min.js`, `css/sweetalert.min.css`, `js/cookie-consent.js`, `css/cookie-consent.css`.

---

## ВАЖНО: что нельзя ломать

### Структура и идентификаторы
Сохранить существующие **ID/классы, на которые завязаны скрипты**:
- `#overlay`
- `#modalForm`
- `#closeModal`
- `#showFormButton`
- калькулятор: `#vizit`, `#oplata`, `#qty1`, `#qty2`, `#grandTotal`
- слайдер: `.slider`, `.slide`, `.dot` (и их логика переключения)
- lightbox: `#lightbox`, `#close-lightbox`, `#prev-slide`, `#next-slide`, `#lightbox-img`

### Контент и SEO
- Не переписывать смысл текстов (можно корректировать оформление и переносы, но не “маркетинг”).
- Не удалять meta-теги, canonical, og-теги, favicon.
- Не удалять счётчик Яндекс.Метрики и `schema.js`.
- Не менять `title`, `description`, `keywords` (если нужно — только структурировать/перенести в более аккуратный вид без изменения).

### Технологические ограничения
- Без Bootstrap/Tailwind/React и других фреймворков.
- Разрешены: чистый CSS (или CSS-переменные), минимальные изменения HTML/PHP-шаблонов.
- Максимально переиспользовать существующую палитру (`--accent` и т.п.), но допускается “осовременить” её оттенки.

## Что сделать (обязательные задачи)

### 1) Визуальная система
Сделать современный “сервисный” дизайн:
- Чёткая типографика, нормальные интервалы, читаемость на мобильных.
- Единая система отступов (например 4/8/12/16/24/32/48).
- Единые радиусы/тени/бордеры (использовать CSS variables).
- Светлый фон + акцентный цвет; не перегружать градиентами.

**Результат:** обновить `:root` переменные и привести стили к единой системе.

### 2) Шапка (header)
Сделать шапку более современной и удобной:
- Визуально выделить телефон и кнопку “Заказать звонок”.
- Сделать адаптив: на ширине до 768px — компактная шапка без “разъезжания” элементов.
- Желательно: добавить лёгкую “липкость” (sticky) и/или полупрозрачный фон на скролле, **если не ломает верстку**.

**Результат:** правки в `header.php` (минимальные) + стили.

### 3) Hero-блок
Сейчас фон тяжёлый и текст иногда теряется.
Сделать:
- Чёткий контраст текста на фоне (оверлей, градиент, блюр-подложка — на ваш вкус).
- Улучшить иерархию: H1, подзаголовок, короткое описание.
- Карточки преимуществ привести к единому виду: одинаковая высота заголовков, аккуратные иконки, одинаковые отступы.

**Результат:** улучшенный hero + карточки без визуального “шума”.

### 4) Блок “О нас” + “Галерея”
Сделать более “профессиональную” подачу:
- Две карточки в ряд на десктопе, одна под другой на мобиле.
- Изображение “автовышка” аккуратно вписать: ограничить размер, добавить рамку/фон/тень.
- Галерею/слайдер не ломать, но улучшить визуально: точки, отступы, рамка/скругление у изображений.

### 5) Контакты + калькулятор
- Разделить визуально две колонки (на десктопе) и сделать понятным CTA: телефон, мессенджеры.
- Калькулятор оформить как “карточку формы”: ровные поля, одинаковая высота input/select, фокус-стили.
- Итоговую стоимость (`#grandTotal`) сделать заметной.

### 6) Подвал (footer)
- Привести к единому стилю: отступы, ссылки, контраст.
- Исправить “маржин-трюк” с отрицательным `margin-top`, если получится сделать нормально через layout (желательно).

## Что можно менять в файлах

Разрешено:
- `header.php`, `footer.php`, `index.php` — **минимальные** правки структуры для лучшего layout (оборачивающие контейнеры, дополнительные классы, но не ломая IDs).
- `css/styles.css` — основная зона редизайна.

Запрещено:

- Удалять подключение скриптов/стилей, метрики, schema.

## Приёмка (Acceptance Criteria)

Считаем задачу выполненной, если:
1) Главная страница выглядит современно и единообразно на ширинах 360 / 768 / 1024 / 1440.
2) Кнопка “Заказать звонок” открывает модалку как и раньше, затемнение работает.
3) Слайдер и lightbox работают как и раньше.
4) Калькулятор считает как и раньше, поля не “ломаются” на мобиле.
5) Вёрстка не уезжает, нет горизонтального скролла на 360px.
6) Не удалены SEO/meta/метрика/каноникал/og.

## Технические подсказки (не обязательно, но удобно)

- Введите переменные:
  - `--space-1..--space-6`
  - `--font-base`, `--font-size-*`
  - `--border`, `--shadow-sm/md/lg`
- Переиспользуйте контейнер:
  - `.container { max-width: var(--container); margin: 0 auto; padding: 0 20px; }`
- Для карточек:
  - `.card { background: var(--surface); border: 1px solid rgba(...); border-radius: var(--radius); box-shadow: var(--shadow); }`
